The BPMN2Asbru Project

< BPMN2Asbru: Condition Wrapper for Clinical Practice Guidelines >
Team  Christian Hinterer
 
Contact Person  Christian Hinterer / e0927843@student.tuwien.ac.at
 
Project  The aim of the BPMN2Asbru project was do develop a parser logic that finds condition terms in clinical practice guidelines and translates them into the Asbru language.
Finding conditions in plain-texts is a very complex task itself, therefore the plain-text clinical practice guidelines have to be modeled in the business process model and notation first.
After that pre-modelling, the terms that contain the assumed conditions can be found easily since they are marked with the xml tag "ConditionExpression".
The key requirement to the BPMN2Asbru parser is to find logical conditions in these text phrases and to translate them into the Asbru languate, which is a completely automatic process- and executeable language, based on the XML standard.
Finding semantics in plain-text phrases, which belongs to the field of information extraction, is a very complex task and the results are often more arguable than sharp.
To handle the input texts the GATE software is used. This software offers a comfortable JAVA API to analyse texts and to create annotations on them, using different sources and dictionaries. It also offers the usage of a plug-in for MetaMap, which is a programm to identify medical concepts in free texts. Using this JAVA API it is possible to send text content to a MetaMap server, either a local one or a remote one, that processes the input using a Metathesaurus for the medical domain (the unified medical language system) hostet by the U.S. national library of medicine.
Based on the annotations made on the texts, the parser logic tries to interpret them correctly and to build logical conditions out of them.
At last, the found conditions are translated into the Asbru language and printed into a file for further usage.
